## 1. 产品概述

### 1.1 产品一句话

> 上传你的资料（PDF、图片、文档、旧网站链接），AI 自动分析你的业务，生成一个面向搜索引擎优化的专业展示官网，并让访问者可以通过表单、邮箱、WhatsApp、微信等方式联系你。

### 1.2 背景与机会

#### 1.2.1 行业现状

大量个人和企业缺乏一个专业的展示官网，或者现有网站质量低、难以被搜到：

- 中小企业没有官网，或网站多年不更新，内容像公司宣传册而非获客工具。
- 自由职业者和工作室靠口头推荐获客，没有一个可以发给他人的专业展示页。
- 外贸工厂的网站英文表达弱、SEO 基础差，海外采购商搜不到。
- 服务商（设计公司、咨询、装修等）的网站无法体现专业能力和案例。
- 现有建站方案要么技术门槛高（WordPress 需要懂主机、插件、SEO），要么模板千篇一律，要么需要大量手动填写内容。
- 随着 ChatGPT、Perplexity、Google AI Overview 等 AI 搜索引擎崛起，"被 AI 引用"成为新的流量入口，但现有工具对此没有准备。

#### 1.2.2 产品机会

ForgeSite 不卖"拖拽建站"，而是卖"展示官网的自动化生成"：

- 自动整理用户已有的分散资料（PDF、Excel、图片、旧网站），降低启动成本。
- AI 自动理解业务，生成专业内容和结构，降低内容门槛。
- 自动做 SEO/GEO，提高被搜索引擎和 AI 推荐的概率。
- 灵活配置联系渠道，让访问者能通过他们习惯的方式联系你。
- 帮助用户建立一个可搜索、可分享、可转化的线上展示资产。

### 1.3 产品定位

#### 1.3.1 是什么

- **AI 展示官网生成器**：上传资料 → AI 理解你的业务 → 自动生成专业官网。
- **面向搜索引擎的内容系统**：每个页面天然适合 Google 和 AI 搜索收录。
- **灵活的展示平台**：适用于企业、个人、工作室、服务商等多种场景。
- **资料结构化工具**：把分散的素材整理成结构化的展示内容。

#### 1.3.2 不是什么

- 不是 Shopify 替代品：不做购物车、在线支付、库存系统。
- 不是 WordPress 替代品：不强调插件生态和复杂 CMS。
- 不是拖拽建站器：用户管理结果而非系统。
- 不是模板市场：不以模板数量作为核心竞争力。

### 1.4 核心价值主张

**对所有人：**

- 上传现有资料即可生成专业官网，不需要自己写文案、不需要排版、不需要学工具。
- 生成的网站天然适合搜索引擎和 AI 搜索。
- 访问者可以通过他们习惯的方式联系你（表单、邮箱、WhatsApp、微信等）。

**对企业/机构：**

- 专业的公司形象展示，包含产品/服务、能力、案例、团队、资质等模块。
- 可用于搜索引擎、社交媒体、展会、名片等所有触点。

**对个人/工作室：**

- 快速生成作品集或个人品牌展示页。
- 分享一个链接就能让对方了解你。

**对服务商/代理商：**

- 可批量为客户交付展示网站，提高交付效率。

## 2. 产品架构：AI 驱动的动态结构推荐

### 2.1 核心理念

ForgeSite 不预设任何场景包。**AI 分析用户上传的资料后，自主判断应该生成什么样的网站结构。**

```
用户上传资料 + 基本信息
        ↓
AI 分析素材内容、识别实体类型
（这是什么类型的展示需求？有什么内容？）
        ↓
AI 推荐网站结构
（首页放什么、需要哪些页面、每个页面用什么模块）
        ↓
用户确认结构，可调整增删
        ↓
AI 生成内容 → SEO/GEO → 发布
```

不需要"选择你的类型"，不需要预设的"企业版/个人版"。同一个引擎，AI 判断适配一切。

### 2.2 AI 结构推荐的工作方式

AI 在分析用户资料后，从三个层面做出判断：

**第一层：实体理解**

AI 识别"这是谁"：

| 识别结果 | 判断依据 |
|---|---|
| 企业（制造/贸易/科技...） | 资料包含公司介绍、产品目录、营业执照、组织架构 |
| 个人品牌/自由职业 | 资料以个人作品、个人简介为主 |
| 服务商/咨询 | 资料以案例、服务说明、团队介绍为主 |
| 其他 | AI 根据内容特征自动归类 |

**第二层：内容分析**

AI 识别"有什么内容可用"：

| 内容类型 | 识别依据 |
|---|---|
| 产品/商品 | 产品目录、规格表、价目表、产品图片 |
| 服务项目 | 服务清单、流程说明、服务案例 |
| 作品/案例 | 项目展示、案例描述、作品图片 |
| 资质证书 | 证书图片、认证文件 |
| 团队成员 | 团队介绍、个人履历 |
| 能力/设备 | 设备清单、产线说明、技术参数 |
| 客户评价 | 推荐信、评价截图 |
| 行业知识 | 行业文章、技术博客 |
| 联系方式 | 邮箱、电话、社交链接 |

**第三层：结构推荐**

基于前两层分析，AI 推荐网站结构：

- **页面列表**：需要哪些页面
- **每个页面的模块组合**：首页放 Hero + 产品预览 + 资质 + CTA，还是 Hero + 作品集 + 关于 + 联系
- **模块排序**：什么顺序最合理
- **内容填充**：每个模块用什么资料来生成

### 2.3 用户确认环节

AI 推荐结构后，进入用户确认环节：

- 展示推荐页面列表（"AI 建议您的网站包含以下页面："）。
- 每个页面展示推荐模块。
- 用户可：✅ 确认 / ❌ 移除某个页面 / ➕ 添加其他页面 / 🔄 调整顺序。
- 确认后进入内容生成阶段。

**关键设计原则：不要一次性展示太多决策选项给用户。AI 推荐的是"最佳猜测"，用户只需微调而不是从零搭建。**

### 2.4 为什么这样更好

| 对比 | 预设场景包 | AI 动态推荐 |
|---|---|---|
| 灵活性 | 只能覆盖预设的类型 | 覆盖无限类型 |
| 维护成本 | 每新增一种类型都要配场景包 | 零维护 |
| 用户体验 | "请选择您的类型" | "我们帮您分析好了" |
| 扩展性 | 受限于预设组合 | 随 AI 能力自然扩展 |

## 3. 目标用户

### 3.1 核心用户画像

#### 画像 A：中小企业主 / 创业公司

| 属性 | 描述 |
|---|---|
| 身份 | 公司老板、创始人、市场负责人 |
| 现状 | 没有官网，或网站过时无法更新 |
| 诉求 | 快速有一个拿得出手的官网，让客户能找到自己 |
| 痛点 | 不懂建站、没时间学工具、预算有限 |
| 关注点 | 上线速度、专业感、能否被搜到、价格 |
| 典型行业 | 制造业、贸易、科技、本地服务 |

#### 画像 B：外贸 / 出海企业

| 属性 | 描述 |
|---|---|
| 身份 | 外贸业务员、工厂主、跨境电商运营 |
| 现状 | 依赖阿里国际站、展会、老客户转介绍 |
| 诉求 | 有专业英文官网用于背调、开发信和社媒导流 |
| 痛点 | 英文弱、不懂 SEO、资料分散 |
| 关注点 | 英文质量、Google 收录、询盘转化 |
| 典型行业 | 五金、机械、包装、电子配件、家居 |

#### 画像 C：自由职业者 / 个人品牌

| 属性 | 描述 |
|---|---|
| 身份 | 设计师、摄影师、咨询顾问、教练 |
| 现状 | 靠熟人推荐，没有一个可展示自己的页面 |
| 诉求 | 快速生成个人品牌展示页，用于社交分享 |
| 痛点 | 不想花时间做网站、不想花钱请人做 |
| 关注点 | 设计感、移动端展示、联系转化 |
| 典型领域 | 设计、摄影、咨询、培训 |

#### 画像 D：服务商 / 代理商（第二期重点）

| 属性 | 描述 |
|---|---|
| 身份 | 建站公司、设计公司、代运营团队 |
| 现状 | 为客户交付网站，重复劳动多 |
| 诉求 | 提高交付效率，批量服务客户 |
| 痛点 | 每个客户都要重新整理资料、写内容 |
| 关注点 | 多站点管理、白标、人效提升 |

### 3.2 MVP 优先服务

MVP 阶段优先服务**画像 A（中小企业）和画像 B（外贸企业）**，因为：

- 这两个群体的付费意愿和付费能力最强。
- 他们对"展示官网"的需求最明确、最完整。
- 他们的问题（没有官网 / 官网质量低）最痛。

画像 C（个人品牌）在 MVP 阶段通过企业版的轻量配置即可覆盖（裁减企业版页面，保留核心模块），不需要额外开发。

## 4. MVP 范围

### 4.1 MVP 核心链路

```
上传资料 → AI 解析 → AI 信息提取 → AI 推荐网站结构 → 
用户确认结构 → 生成网站 → 预览编辑 → 
配置联系方式 → 发布上线 → 追踪访问和转化
```

### 4.2 MVP 功能清单

| 功能模块 | MVP | 说明 |
|---|---|---|
| 素材上传与解析 | ✅ | PDF、Excel、Word、PPT、图片、URL |
| AI 信息提取 | ✅ | 公司/个人、产品/服务、资质、联系方式 |
| 用户确认 | ✅ | 表单审查、置信度显示、来源追踪 |
| 网站生成 | ✅ | 5 分钟首版生成 |
| 页面模块编辑 | ✅ | 修改内容、替换图片、重新生成 |
| SEO 配置 | ✅ | Title、Meta、Sitemap、Robots、结构化数据 |
| GEO 内容 | ✅ | DirectAnswer、FAQ、Evidence、JSON-LD |
| 联系方式配置 | ✅ | 表单、邮箱、WhatsApp、微信、电话、LinkedIn |
| 发布（子域名） | ✅ | customer.forgesite.com |
| 数据看板 | ✅ | 访问、转化事件 |
| 询盘管理 | ✅ | 查看、导出 |
| 自定义域名 | ❌ | 第二期 |
| 多语言 | ❌ | 第二期 |
| 多模板选择 | ✅ | 至少 3 套风格 |
| 网站结构确认 | ✅ | AI 推荐 + 用户确认 |

### 4.3 不在 MVP 范围

- 在线支付 / 购物车
- 拖拽建站器
- 多业务员权限
- 客户登录门户
- 营销邮件群发
- 广告投放管理
- 插件市场
- 复杂 CRM
- ERP 对接
- 完整版本树管理

## 5. 功能需求详述

### 5.1 素材上传与解析

#### 5.1.1 支持素材类型

**MVP 必须支持：**

| 类型 | 来源 | 用途 |
|---|---|---|
| PDF | 产品目录、公司介绍、资质证明 | 文本提取、结构分析 |
| Excel | 产品表、服务价目表 | 表格结构识别 |
| Word | 公司/个人介绍 | 文档内容提取 |
| PPT | 公司介绍、方案展示 | 内容提取 |
| 图片 | 产品图、现场图、作品图、证书 | OCR 识别、图库 |
| URL | 旧官网、社交主页 | 内容抓取 |

**参数限制：**
- 单次上传 ≥ 50 个文件
- 单文件最大 50MB
- 上传后显示解析状态：等待中、解析中、已完成、失败

#### 5.1.2 解析链路

```
上传 → 文件校验 → 格式识别 → 文本提取 →
结构化内容分块 → 图片分类 → 元数据提取 → 
生成解析报告
```

- 异步解析，上传完成即可离开页面。
- 解析失败不影响其他文件。
- 解析结果保留来源信息（页码、行号、文件名）。

#### 5.1.3 文件管理

- 素材按类型分组展示（产品资料、公司介绍、图片、证书等）。
- 支持重新上传和删除。
- 显示每个文件的解析状态和提取数量。

### 5.2 AI 信息提取

#### 5.2.1 提取字段

**基本信息（通用）：**

| 字段 | 说明 | 置信度要求 |
|---|---|---|
| 名称 | 公司名/个人品牌名 | 高 |
| 简介 | 一句话介绍做什么 | 中 |
| 联系方式 | 邮箱、电话、地址 | 高 |
| 社交账号 | WhatsApp、微信、LinkedIn 等 | 高 |
| 核心优势 | AI 提炼的 2-5 条核心卖点 | 中（可编辑） |

**产品/服务信息：**

| 字段 | 说明 |
|---|---|
| 名称 | 产品/服务名称 |
| 分类 | 所属类别 |
| 描述 | AI 生成的英文/中文描述 |
| 规格参数 | 关键键值对（如材质、尺寸、MOQ、价格区间） |
| 图片 | 匹配的产品/服务图片 |
| 应用场景 | 适合什么场景使用 |
| 可定制项 | 定制化能力 |

**资质/认证信息：**

| 字段 | 说明 |
|---|---|
| 证书名称 | ISO、CE、专利等 |
| 颁发机构 | 认证机构名称 |
| 有效期 | 证书有效时间 |

#### 5.2.2 提取规则

- 输出必须走 JSON Schema / Structured Output。
- 每个字段保存来源引用（文件 ID、页码、文本片段）。
- 关键事实字段（认证、年份、产能、出口国家等）必须有来源，低置信度必须标记"待确认"。
- 不允许没有来源的关键事实直接写入最终网站。
- 提取结果以表单方式展示，用户可修改。
- 支持一键接受全部高置信度 AI 建议。

### 5.3 网站生成

#### 5.3.1 生成流程

```
用户确认资料 → 选择场景类型 → 选择模板 →
AI 生成页面内容 → 渲染为静态 HTML →
输出到预览环境
```

- 用户确认资料后，5 分钟内生成首版网站。
- 生成语言默认为中文（管理端）/ 英文（前台），用户可选择。
- 用户可选择至少 3 套风格模板。
- 页面可预览桌面端和移动端。

#### 5.3.2 生成策略

- AI 根据用户资料分析结果自主决定页面结构和模块组合（详见第 7 章）。
- AI 基于用户资料库生成每个模块的内容。
- 模板控制视觉风格和布局，不直接复制文案。
- 同一场景下的多个网站不能生成高度相似的内容。

### 5.4 简易编辑器

#### 5.4.1 编辑能力

MVP 支持轻量编辑，不做复杂拖拽：

- 修改基本信息（公司名、简介、联系方式等）。
- 修改产品/服务信息。
- 替换图片。
- 调整产品/服务分类。
- 修改 SEO 标题和描述。
- 隐藏或显示页面模块。
- 重新生成某一段内容（如产品描述、FAQ）。
- 支持恢复到上一个版本。

#### 5.4.2 不做

- 不做自由拖拽页面编辑器。
- 不做复杂动画编辑。
- 不做 CSS/JS 自定义。

### 5.5 发布与域名

#### 5.5.1 MVP 发布方式

- 免费预览/发布域名：`customer.forgesite.com`
- 点击发布后 2 分钟内网站可访问。
- 支持草稿站和正式站。
- 支持重新发布。
- 支持查看发布时间和版本。

#### 5.5.2 发布产物

每次发布生成不可变版本：

```
index.html
products/index.html
products/{slug}/index.html
about/index.html
contact/index.html
sitemap.xml
robots.txt
assets/*
```

### 5.6 数据看板

#### 5.6.1 MVP 指标

- 总访问量。
- 页面访问量（Top 页面）。
- 访问国家/地区分布。
- 联系渠道点击次数（邮箱、WhatsApp、微信等）。
- 表单提交数。
- 热门页面排名。

#### 5.6.2 数据展示

- 最近 7 天 / 30 天切换。
- 表单提交可在后台查看详情。
- 支持邮件通知新表单提交。
- 支持导出询盘 CSV。

## 6. 页面模块库

### 6.1 模块定义

ForgeSite 的页面由模块组合而成。AI 根据用户资料分析和行业理解自动选择模块组合，用户也可以手动调整。

| 模块名称 | 描述 | 适用场景 |
|---|---|---|
| Hero | 首页主视觉：标题、副标题、CTA 按钮 | 所有 |

## 7. AI 网站结构推荐机制

### 7.1 推荐流程

```
用户上传资料
  ↓
[阶段 1] 素材分析
  AI 解析所有文件，理解内容类型和结构
  ↓
[阶段 2] 实体识别
  AI 判断展示主体类型（企业/个人/服务商/其他）
  ↓
[阶段 3] 内容盘点
  AI 识别有哪些可用内容（产品、服务、作品、资质、团队...）
  ↓
[阶段 4] 结构推荐
  AI 推荐页面列表 + 每个页面的模块组合 + 模块排序
  ↓
[阶段 5] 用户确认
  用户确认推荐结构（可增删调整）
  ↓
进入内容生成
```

### 7.2 AI 推荐示例

├── 工厂能力      设备清单 + 产线流程 + 质量控制
├── 资质证书      证书展示（带颁发机构和有效期）
├── 关于我们      公司简介 + 发展历程 + 出口市场
└── 联系          表单 + WhatsApp + 邮箱 + LinkedIn
```

**示例 2：摄影师**

```

### 7.3 推荐策略

**AI 判断页面重要性的规则：**

- 用户资料中内容最丰富的类别 → 作为主要页面（如产品多则产品页优先）。
- 行业特定的信任要素 → 自动推荐（如制造业推荐能力页、资质页）。
- 非必要但能增强信任的页面 → 作为"建议添加"而非默认包含（如团队、博客）。

**用户修改推荐的方式：**

- 移除不需要的页面（"我不需要博客"）。
- 添加模块到已有页面（"首页再加一个客户评价"）。
- 调整模块顺序。
- 所有修改不影响已生成的内容结构。

### 7.4 推荐置信度与退路

- AI 对推荐的每个页面/模块有置信度标记。
- 高置信度（> 80%）：直接推荐给用户，默认勾选。
- 中置信度（60-80%）：推荐，标注"建议"。
- 低置信度（< 60%）：建议用户补充资料后重试。
- 当资料信息严重不足时，提供"快速向导"模式（几步简单问答代替资料上传）。

### 9.1 标准 SEO（所有站点自动配置）

- Open Graph / Twitter Card 标签。
- Canonical 标签。

### 9.2 GEO（Generative Engine Optimization）

GEO 是 ForgeSite 的核心差异化能力——面向 AI 答案引擎（ChatGPT、Perplexity、Google AI Overview、Kimi、豆包等）的内容优化。

#### 9.2.1 GEO 内容单元

每个核心页面自动生成以下 AI 可读内容单元：

| 单元 | 说明 | 必选 |
|---|---|---|
| DirectAnswer | 页面顶部 40-80 词的直接答案，说明该页面解决什么问题 | ✅ 核心页 |
| EntitySummary | 清晰标注业务实体（名称、类型、行业、地域等） | ✅ 所有页 |
| EvidenceBlock | 带来源的关键事实块（资质、数据、能力等） | ✅ 有据可查时 |
| BuyerQuestion | 围绕真实采购/选择问题生成的 FAQ | ✅ 产品/服务页 |
| ComparisonBlock | 产品/服务/方案对比 | 可选 |
| HowToChoose | 选型/选择建议 | ✅ 产品/服务页 |
| ApplicationBlock | 应用场景说明 | 可选 |
| CtaBlock | 转化引导 | ✅ 所有页 |

#### 9.2.2 结构化数据（JSON-LD）

自动注入：

- `Organization` / `Person` — 所有站点
- `Product` / `Service` — 产品/服务页
- `FAQPage` — 有 FAQ 的页面
- `BreadcrumbList` — 所有页面
- `ImageGallery` — 有图库的页面
- `LocalBusiness` — 有实体地址的站点

#### 9.2.3 GEO 质量评分

每个核心页面生成 GEO Score（0-100）：

| 分数区间 | 状态 | 操作 |
|---|---|---|
| 80-100 | ✅ 良好 | 允许发布 |
| 60-79 | ⚠️ 需优化 | 允许发布，提示优化项 |
| < 60 | ❌ 不建议发布 | 要求用户补充资料或重新生成 |

评分维度：

- 实体清晰度
- 答案完整度
- 证据充分度
- 结构化程度
- 差异化程度
- 转化明确度

#### 9.2.4 AI 防幻觉规则

关键事实类字段（认证、年份、产能、数据等）必须满足至少一个条件：
- 来自用户确认
- 来自上传文件
- 来自用户输入

AI 可自由生成但需要符合风格的字段：
- 营销描述
- FAQ 表达方式
- 采购/选择建议
- SEO Title / Meta Description
- DirectAnswer 摘要

## 10. 核心用户流程

### 10.1 新建站流程

```
1. 注册并创建项目
2. 输入基本信息（名称、行业/领域、一句话简介）
3. 上传素材（PDF、Excel、图片、URL...）
4. 系统异步解析素材
5. AI 生成结构化信息
6. AI 推荐网站结构（页面列表 + 模块组合）
7. 用户确认 AI 推荐的结构（可增删页面和模块）
8. 用户补充缺失信息
9. 用户选择网站风格模板
10. AI 生成首版网站
11. 用户预览并编辑内容
12. 用户配置联系方式（表单、WhatsApp、微信、邮箱等）
13. 用户发布网站（二级域名）
14. 系统开始统计访问和转化数据
```

### 10.2 内容更新流程

```
1. 用户上传新资料（新产品表、新图片等）
2. AI 识别新增和变更内容
3. 用户确认更新
4. 系统增量更新受影响页面
5. 用户重新发布
```

### 10.3 转化处理流程

```
1. 访问者浏览页面
2. 点击联系按钮 / 提交表单
3. 系统记录来源页面和转化事件
4. 站点管理员收到通知（邮件）
5. 管理员在后台查看转化详情
6. 管理员跟进处理
```

## 15. 风险与应对

| 风险 | 等级 | 应对 |
|---|---|---|
| AI 内容编造 | 高 | 关键事实必须有来源；不确定字段标记待确认；发布前要求确认 |
| 生成网站同质化 | 中 | 基于资料差异生成内容；模板控制结构不控制文案；行业特定 prompt |
| 用户期待立刻有流量 | 中 | 产品内展示收录进度和质量分；强调社媒和私域使用场景 |
| 文件解析质量不稳定 | 中 | OCR + 文本 + 用户确认三层兜底；低置信度提示补充 |
| AI 推荐结构不准 | 中 | 提供推荐置信度标记；用户可自由调整推荐结构；资料不足时引导补充
